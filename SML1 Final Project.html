The Winton dataset contains numerous predictors which we could choose from in building models. However, our test dataset only contains predictors Ret_2 through Ret_120, so while we could include other data from the Winton training dataset, it would be of no value when testing the model on the test dataset. Therefore, we restrict our possible predictors to Ret_2 through Ret_120 (and transformations or derivations of those predictors).

With so many predictors, and a signficiant deal of noise in the predictors, several of the approaches we took in previous attempts at building models (particularly in Assignments 4 and 5) utilized dimensionality reduction and smoothing techniques. The results from these approaches were disappointing, to say the least. A model submitted for Assignment 5 performed worse than the naive submission upon creating our Kaggle account in Assignment 1. The model used principal components analysis (PCA) to attempt to reduce the number of predictors, and then used smoothing splines on the factors that explained 95% of the total variance as predictors for a general additive model (GAM).

Following the disappointing performance of the GAM model using smoothed PCA factors as predictors, we considered other transformations of predictors. One thing we noticed was that in order to explain 95% of the total variance with PCA factors, we were not able to reduce the dimensionality to us. This suggested that there is a high degree of noise-to-signal in the predictors, and furthermore, there is not a small subset of the predictors that is still representative of substantially all of the signal. This realization is what led to our usage of random forest models, as these models should perform well and avoid overfitting when we have a large number of variables that are not correlated/redundant (as these seem not to be, otherwise PCA ought to have been more successful in reducing dimensionality).

One transformation we considered across models in an effort to work around the low signal-to-noise problem was simply considering the sign of the predictor, rather than the actual numerical quantity of the predictor. That is, we wrote a function that transformed our training set to take the value 1 if any given element was positive, and 0 otherwise. We then used this dataset as our training data for several models. This transformation appeared to retain quite a bit of informational value; for instance, when comparing our random forest model on the untransformed training set to the random forest model fit to the binary training set, the binary random forest model performed worse, but not extremely so. In fact, it managed to explain roughly 2/3rds of the variance that the random forest fit to the untransformed dataset did (10% vs 16% variance explained, respectively, as can be seen below in the Random Forest section). Ultimately, we elected to proceed with the random forest model fit to the untransformed dataset, but it was interesting that the models were not as far apart as one might initially expect.

ConvertSubmission = function(mypreds, outfname="mysubmission.txt", 
      fullrowfile = "http://www.stat.cmu.edu/~cschafer/MSCF/fullrowtest.txt")
{
  fullrowtest = scan(fullrowfile,sep=",",quiet=TRUE)

  if(length(mypreds) != sum(fullrowtest))
  {
    stop("Your vector of predictions is not the correct length.")
  }
  if(mode(mypreds) != "numeric")
  {
    stop("Your vector of predictions is not of the correct type.")
  }

  write.table("Id,Predicted",file=outfname,sep=",",row.names=F,col.names=F,quote=F)
  pos = 1
  for(i in 1:length(fullrowtest))
  {
     rowlabels = paste(i,"_",1:62,sep="")
     holdout = data.frame(rowlabels,rep(0,62))
     if(fullrowtest[i] == 1)
     {
       holdout[61,2] = mypreds[pos]
       pos = pos + 1
     }
     write.table(holdout, file=outfname, sep=",", 
          row.names=F,col.names=F,quote=F,append=TRUE)
   }
}

Random Forest Model¶

In [32]:
library(randomForest)
set.seed(17)
randformod = randomForest(modform, data=clean_train, ntree=300, importance=TRUE)

We are choosing the random forest model fit on the entire set of untransformed predictors to be our final model. We chose the random forest for several reasons. First, random forests tend to be fairly resilient to the perils of overfitting. Overfitting is a significant concern on this dataset due to the low signal-to-noise ratio. If we have a model that overfits on the training data, it will be fitting too much of the noise, and will likely have very poor out-of-sample performance. In fact, we believe this is partially what contributed to the poor performance of our GAM model when it was submitted to Kaggle. So, in choosing a random forest, we hope the methodology itself will help prevent overfitting. Additionally, by using untransformed predictors, we are keeping the model "simple". Stacking models on models might be intellectually satisfying, but again, our concerns around the amount of noise in the predictors suggests to us that we should endeavor to keep models simple. Below we present diagnostic plots for the model, and we will comment below each plot.
